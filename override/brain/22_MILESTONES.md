# Milestones

Milestones define accepted phases, dependencies, and completion requirements.

## Milestone 0 — Brain Constitution

Goal: Establish permanent product, architecture, engineering, and roadmap documentation.

Acceptance criteria: Brain structure exists; product bible exists; authority hierarchy exists; current state is separated from product/architecture; ADR log exists.

## Milestone 1 — Runtime Foundation Reality

Goal: Make lifecycle, DI, modules, events, logging, health, and shutdown reliable.

Dependencies: Brain Constitution.

Completion requires automated tests, lifecycle recovery, event contract validation, and current-state update.

## Milestone 2 — Workspace Observation Baseline

Goal: Reliable active window, running apps, screen, clipboard, project, browser tab, and service observation on the primary OS.

Completion requires real data, not mock-only signals.

## Milestone 3 — Provider Bridge

Goal: Wrap legacy browser, desktop, file, shell, and git actions as governed providers.

Completion requires permission model, risk levels, schemas, verification hooks, and integration tests.

## Milestone 4 — World Model and Session Recall

Goal: Persist workspace graph and restore sessions.

Completion requires entity graph persistence, session summaries, context switching, and deletion controls.

## Milestone 5 — Reasoning and Planning Integration

Goal: Reason over context, goals, memory, risk, and providers to produce plans.

Completion requires explainability, confidence, approvals, and planner contracts.

## Milestone 6 — Adaptive Product Workspaces

Goal: Dynamic UI for coding, research, learning, meetings, career, and diagrams.

Completion requires task-specific information hierarchy and anti-dashboard compliance.

## Milestone 7 — Long-term Goal and Behavior Intelligence

Goal: Connect memory, behavior, goals, and workspace context into useful long-term assistance.

Completion requires advisory behavior only, interruption policy, and user controls.
