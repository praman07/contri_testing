# World Model

The World Model is Override's persistent representation of the user's digital environment and work over time.

## Entities

Entities include users, goals, projects, repositories, files, documents, browser tabs, applications, windows, services, containers, meetings, messages, plans, providers, actions, verification results, and memories.

## Relationships

Relationships describe work: file belongs to project, tab supports research, command modified repository, meeting created task, goal depends on project, provider executed action, verification confirmed outcome.

## Persistence

Short-term context is kept in working memory. Durable world state is persisted locally with user control, redaction, retention policies, and deletion capability.

## Context evolution

The World Model changes as observations, perception frames, provider results, user approvals, verification reports, and memory consolidation events arrive.

## Workspace graph

The workspace graph lets Override understand how windows, files, apps, tabs, goals, and tasks relate, enabling session recall and context switching.

## Reasoning inputs

Reasoning consumes a summarized, privacy-filtered slice of the World Model relevant to the active goal.
