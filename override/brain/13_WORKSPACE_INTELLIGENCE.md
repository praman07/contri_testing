# Workspace Intelligence

Workspace Intelligence is the subsystem that turns raw computer state into an understanding of active work.

## Tracked workspace elements

Active windows, applications, processes, projects, Git repositories, browser tabs, documents, Docker containers, services, terminals, meetings, messages, calendars, files, tasks, and goals.

## Project awareness

Override identifies project roots, dependencies, scripts, tests, services, architecture docs, branches, recent commits, and open changes.

## Browser and document awareness

Browser tabs and documents are linked to tasks, research claims, forms, meetings, and applications when permission allows.

## Services and Docker

Running services, ports, logs, containers, and health checks are part of the workspace model when relevant to development or operations.

## Workspace summaries

A workspace summary answers: what is active, why it matters, what changed, what is blocked, what can be done next, and what requires approval.

## Session restoration

On return, Override restores goals, project state, open work, recent actions, verification state, and the next recommended step.

## Context switching

When focus changes, Override preserves the previous context and loads the new one without losing continuity.

## Multi-project awareness

Override can track several projects simultaneously while keeping goals, memories, branches, services, and documents separated.
