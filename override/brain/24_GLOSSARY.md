# Glossary

Agent: A bounded cognitive worker that performs a scoped task under orchestration.

Agentic Operating Layer: A cognitive layer above the host OS that understands context, coordinates tools, and executes approved work.

Capability: A declared action a provider can perform.

Context: The current relevant state needed to understand active work.

Execution: Running approved actions through providers.

Goal: A user objective tracked over time.

Knowledge: Consolidated reusable understanding distilled from memory and verified outcomes.

Memory: Stored past experiences, facts, preferences, and procedures.

Observation: Raw signal capture from environment, apps, devices, or user activity.

Perception: Structured interpretation of observations.

Planner: Subsystem that turns reasoning results into executable plans.

Provider: Controlled adapter that exposes external capabilities such as browser, desktop, file, shell, git, Docker, or communication.

Reasoning: Evaluation of context, goals, risks, memory, and provider options to decide what should happen next.

Session: A bounded period of work with goals, context, actions, decisions, and outcomes.

Verification: Evidence-based confirmation that an action or plan achieved its intended result.

Workspace: The active digital environment: projects, windows, apps, tabs, docs, services, meetings, files, and goals.

Workspace Intelligence: Understanding of workspace entities and relationships across time.

World Model: Persistent graph of entities, relationships, context, memory, and work state.
