# Memory and Knowledge

Memory gives Override continuity. Knowledge turns repeated experience into reusable understanding.

## Episodic memory

Records specific events: sessions, actions, decisions, errors, approvals, and outcomes.

## Semantic memory

Stores stable facts: user preferences, project facts, architecture decisions, domain concepts, and recurring constraints.

## Procedural memory

Stores workflows: how a task was successfully completed, provider sequences, checks, and recovery strategies.

## Long-term memory

Long-term memory is local-first, transparent, searchable, editable, and forgettable. It must respect retention and privacy policies.

## Knowledge extraction

Knowledge is distilled from verified outcomes, repeated patterns, user corrections, and explicit preferences.

## Session recall

Recall reconstructs work context: goals, files, tabs, commands, decisions, failures, verification, and next steps.

## User preferences

Preferences are facts with provenance and confidence. The user can inspect, edit, or delete them.
